The Engineering Brain
for AI Development

Ortho provides deep repository understanding, persistent engineering memory, and intelligent workflow orchestration. The LLM becomes a more powerful tool with Ortho as its context engine.

The Problem with AI Coding Today

LLMs are powerful at generation, but they lack the engineering intelligence to generate correctly.

🤔 AI Forgets Context

Without repository understanding, every prompt starts from scratch. No continuity, no institutional memory.

🧠 Hallucinates Architecture

LLMs can invent patterns that don't exist. They make confident statements about structure they've never seen.

⚡ Repeats Mistakes

Without analyzing what failed before, the same bugs resurface. Each task starts without learned patterns.

⚠️ No Engineering Memory

Decisions, ADRs, and lessons learned vanish. Next time? Same decisions get made again.

✓ How Ortho Solves It

  • • Deep Repository Scanning: Symbols, imports, call graphs, architecture patterns
  • • Persistent Context Hub: Every artifact, decision, and lesson stored and searchable
  • • Architectural Intelligence: Auto-detect patterns, dependencies, blast radius, technical debt
  • • Evidence-Driven Workflows: Every decision backed by test output, verification, human approval

Five Pillars of Intelligence

Each pillar is independently useful but more powerful together.

📚 Repository Intelligence

Deep understanding of repository structure through AST analysis and graph construction.

  • Symbol extraction
  • Call graphs
  • Import analysis
  • Module detection
  • Incremental indexing

🎯 ContextHub

Persistent knowledge store with multi-modal search (full-text + semantic + hybrid).

  • Artifact storage
  • BM25 search
  • Semantic search
  • Hybrid retrieval
  • Git metadata

🏗️ Architectural Intelligence

Automatic detection of architecture styles, layers, subsystems, and dependency health.

  • Pattern detection
  • Layer extraction
  • Subsystem discovery
  • Impact analysis
  • Debt scoring

⚙️ Engineering Orchestration

Intelligent coordination of engineering workflows using ASES methodology.

  • Workflow automation
  • Agent selection
  • Task planning
  • Approval gates
  • Evidence tracking

⚡ Token Optimization

Maximum engineering value per token through intelligent context assembly.

  • Context ranking
  • Deduplication
  • Compression
  • Budget management
  • Quality logging

Command Line Interface

Everything accessible from the terminal. Simple, powerful, scriptable.

$ ortho init
$ ortho scan
$ ortho analyze --impact src/auth/service.py
$ ortho context search "authentication flow"
$ ortho run "add rate limiting to API"
